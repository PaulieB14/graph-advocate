## What it does

Joins four data sources in one call to answer:

> *Which x402 services are **actually being paid** and **actually verified agents** right now?*

| Source | Role |
|--------|------|
| x402 Base subgraph (`Cb56epg3...`) | Real on-chain settlements in the last N hours |
| agent0-base 8004 subgraph (`43s9hQRu...`) | ERC-8004 registrations on Base |
| 8004scan.io | Multi-chain ERC-8004 fallback (Arbitrum, BSC, Celo) |
| CDP x402 Bazaar | Service metadata, pricing, descriptions |

## Request

```
GET /bazaar/active?hours=24&limit=15&q=weather
```

| Param | Default | Notes |
|-------|---------|-------|
| `hours` | `24` | Activity window, max 168 |
| `limit` | `15` | Max results, capped at 50 |
| `q` | — | Optional keyword filter over resource + description |

## Ranking

1. ERC-8004-verified agents first
2. Highest payment count in the window
3. Highest USDC volume
4. Deduped by `payTo` wallet

## Why it's useful

CDP Bazaar has 15k+ listings with no activity signal. The x402 Base subgraph has real-time settlements but no metadata. Joining them gives you a filtered, activity-ranked view of the agent economy — paid, working, verified services you can call today.
